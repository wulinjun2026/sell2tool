# 通用产品销售助手 — 系统配置清单

**版本**: v1.1.0  
**更新**: 2026-06-15  

本文档汇总当前系统中**可配置项**：环境变量、部署参数、模板与数据种子、前端常量及客户端存储键。修改方式分为 **改 `.env`（推荐）**、**改配置文件/模板**、**改源码常量** 三类。

> 配置模板见项目根目录 [`.env.example`](../.env.example)。生产环境务必修改 `AUTH_SECRET`、`MYSQL_PASSWORD`、`PUBLIC_BASE_URL`，并关闭 `AUTH_DEV_MODE`。

---

## 一、环境变量（`.env`）

服务端启动时由 `server/db.js` 加载根目录 `.env`（已存在的环境变量不会被覆盖）。

### 1.1 服务与数据库

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `3000` | Node 监听端口 |
| `NODE_ENV` | — | `production` 时关闭 devCode 等非生产行为 |
| `DB_DRIVER` | `sqlite` | `sqlite`（本地）或 `mysql`（生产） |
| `MYSQL_HOST` | — | MySQL 主机 |
| `MYSQL_PORT` | — | MySQL 端口 |
| `MYSQL_USER` | — | 数据库用户 |
| `MYSQL_PASSWORD` | — | 数据库密码 |
| `MYSQL_DATABASE` | — | 库名，如 `used_car_assistant` |
| `MYSQL_CONNECTION_LIMIT` | `20` | 连接池上限（`server/db/mysqlAdapter.js`） |

### 1.2 登录与鉴权

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `AUTH_SECRET` | `uca-dev-secret-change-in-production` | JWT HMAC 密钥，**生产必改** |
| `AUTH_DEV_MODE` | 未设置 | `true` 时发送验证码接口返回 `devCode` |
| `AUTH_DISABLED` | 未设置 | `true` 时跳过登录（仅本地调试） |
| `AUTH_TEST_USER_ID` | `auth-disabled-user` | `AUTH_DISABLED` 时注入的用户 ID |

**代码内固定（改源码）**：

| 项 | 值 | 文件 |
|----|-----|------|
| JWT 有效期 | 30 天 | `server/services/authService.js` → `TOKEN_TTL_MS` |
| 验证码有效期 | 5 分钟 | `authService.js` → `CODE_TTL_MS` |
| 免费试用天数 | 20 天 | `server/services/userRepository.js` → `DEFAULT_TRIAL_DAYS` |
| 免费产品上限 | 40 个 | `userRepository.js` → `DEFAULT_PRODUCT_LIMIT` |
| 付费升级规则 | 验证码 = 手机号后 7 位 | `authService.js` → `verifySmsCode` |

### 1.3 AI 文案（DeepSeek / OpenAI 兼容）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `VISION_API_KEY` | — | 文本 API Key（别名：`DEEPSEEK_API_KEY`） |
| `VISION_API_URL` | `https://api.deepseek.com/chat/completions` | Chat Completions 地址 |
| `VISION_MODEL` | `deepseek-chat` | 模型名（别名：`DEEPSEEK_MODEL`） |
| `VISION_TIMEOUT_MS` | `45000` | 单次请求超时（毫秒） |
| `SELLING_LLM_STRICT` | 未设置 | `true` 时 LLM 失败**不**降级本地模板 |

**涉及能力**：描述润色、卖点生成、描述字段提取（`copyPolish.js`、`descExtract.js`）。

### 1.4 AI 请求队列

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `AI_QUEUE_CONCURRENCY` | `3` | 同时执行的 AI 任务数 |
| `AI_QUEUE_MAX` | `30` | 排队任务上限，超出返回 `AI_QUEUE_FULL` |
| `AI_QUEUE_WAIT_MS` | `120000` | 排队最长等待（毫秒），超时 `AI_QUEUE_TIMEOUT` |

健康检查 `GET /api/health` 的 `aiQueue` 字段反映当前队列状态。

### 1.5 对外 URL

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PUBLIC_BASE_URL` | — | 对外访问根 URL；用于绝对图片链接、分享场景（勿 trailing `/`） |

### 1.6 上传与 Nginx（部署侧）

| 项 | 建议值 | 说明 |
|----|--------|------|
| Nginx `client_max_body_size` | `10m` | `scripts/deploy-remote.sh` 中已配置 |
| Multer 产品照片上限 | **5 MB** | `server/index.js`（改源码） |
| Multer 二维码上限 | **2 MB** | `server/index.js`（改源码） |

## 五、长图模板配置（`assets/templates/*.json`）

数据库 `poster_templates` 表指向 JSON 路径；空库时种子插入 3 套模板。

| 模板 ID | 名称 | 文件 |
|---------|------|------|
| `tpl_simple_01` | 简约白 | `tpl_simple_01.json` |
| `tpl_business_01` | 商务蓝 | `tpl_business_01.json` |
| `tpl_sport_01` | 运动橙 | `tpl_sport_01.json` |

### 5.1 模板 JSON 主要可配字段

| 路径 | 说明 |
|------|------|
| `canvas.width` | 布局逻辑宽度（通常 750） |
| `canvas.exportMaxWidth` | 正式导出宽度上限（默认 1242） |
| `canvas.exportImageMaxEdge` | 服务端嵌入长边（默认 2880） |
| `canvas.exportMaxBytes` | 导出体积参考上限 |
| `canvas.previewExportWidth` | 可选；预览宽度，缺省 **375**（代码默认） |
| `canvas.previewImageMaxEdge` | 可选；预览嵌入长边，缺省 **480** |
| `theme.*` | 主色、价格色、字号等 |
| `blocks[]` | 区块类型、高度、padding、bind、style |
| `multiVehicle.*` | 多车区块模板与分割线 |

规范详见 [长图layoutSchema规范.md](./长图layoutSchema规范.md)。

### 5.2 服务端长图渲染默认（`posterRender.js`）

| 场景 | 参数 | 默认 |
|------|------|------|
| 预览 | exportWidth | 375 |
| 预览 | embed quality | 72 |
| 正式 | exportWidth | `layout.canvas.exportMaxWidth` 或 1242 |
| 正式 | embed quality | 93 |
| 二维码目标像素 | 预览 / 正式 | ≥320 / ≥768 |

客户端 compose 响应中的 `embed.maxEdge`、`embed.quality` 由上述逻辑计算后下发。

## 六、数据库种子与元数据

### 6.1 内置卖点库

`server/db.js` → `SEED_POINTS`：约 16 条 `selling_point_builtin`（仅库中不存在时插入）。修改需改源码或直接在 DB 中维护。

### 6.2 经销商默认资料

SQLite 首次初始化可写入默认 `dealer_profile`（空库逻辑）；生产多用户下每用户独立 `dealer_profile` 行。

### 6.3 Schema 元数据（SQLite `app_meta`）

| key | 含义 |
|-----|------|
| `schema_version` | 结构版本 |
| `cache_quota_bytes` | 规划缓存配额 `524288000`（500MB，客户端 LRU 为主） |

## 九、配置修改速查

| 需求 | 推荐改法 |
|------|----------|
| 换生产数据库 | `.env` → `MYSQL_*` |
| 换 DeepSeek Key | `.env` → `VISION_API_KEY` |
| 加强 JWT 安全 | `.env` → `AUTH_SECRET`；关闭 `AUTH_DEV_MODE` |
| 调整试用天数/产品上限 | 改 `userRepository.js` 常量 |
| 换长图模板样式 | 编辑 `assets/templates/*.json` 或 DB `poster_templates` |
| 换 APK 默认服务器 | 改 `config.js` → `DEFAULT_SERVER_URL` + 重打包 |
| 浏览器连远程 API | 「我的」页填写服务器地址 → `uca_server_url` |
| 提高 AI 并发 | `.env` → `AI_QUEUE_CONCURRENCY` |
| 新增海报模板 | 新增 JSON + `INSERT poster_templates` + `enabled=1` |
| 关闭登录调试 | 勿设 `AUTH_DISABLED`；生产关闭 `AUTH_DEV_MODE` |

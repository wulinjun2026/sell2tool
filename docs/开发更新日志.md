## v1.1.0 变更摘要（2026-06-15）

### A. 多用户账号与数据隔离

| 项 | 说明 |
|----|------|
| 手机号登录 | `POST /api/auth/sms/send`、`/verify`；验证码 5 分钟有效，60 秒重发间隔 |
| JWT 鉴权 | `Authorization: Bearer`；业务 API 默认 `requireAuth` |
| 用户表 | 迁移 `004_users`：`users`、`auth_codes`；`vehicles` / `dealer_profile` 增加 `user_id` |
| 免费试用 | 注册后 **20 天**、最多 **40 个产品**；超限返回 `PRODUCT_LIMIT_REACHED` / `TRIAL_EXPIRED` |
| 付费升级 | 验证码为手机号**后 7 位**时 `plan=paid`，解除试用与数量限制 |
| 前端 | 「我的」展示试用剩余天数与配额；付费墙对话框 |

**相关文件**：`server/services/authService.js`、`userRepository.js`、`public/js/auth.js`、`app.js`

### B. 产品品牌化与录入文案

| 项 | 说明 |
|----|------|
| 界面名称 | 「通用产品销售助手」；列表为「我的产品」 |
| 上传步骤 | 外观 / **细节** / **补充**（原内饰、座椅文案调整） |
| 登录 UI | 「获取验证码」按钮置于登录按钮正上方；倒计时从 60 秒显示 |

### C. 长图客户端渲染（compose + client PNG）

| 项 | 说明 |
|----|------|
| 新 API | `POST /api/posters/compose` 返回 SVG 文档、嵌入参数、图片 URL（不写 PNG 磁盘） |
| 客户端 | `posterRenderClient.js`：compose → `embedImagesInSvg` → `svgToPngBlob` |
| 嵌入优化 | 6 路并发加载 + LRU 缓存（`posterImageEmbedClient.js`） |
| 进度联动 | `onProgress` 上报 0–98%；`posterProgress.js` 98% 后 98.1–99.9% 等待动画 |

**相关文件**：`public/js/posterRenderClient.js`、`posterProgress.js`、`posterImageEmbedClient.js`、`server/index.js`

### D. 上传与 AI 性能

| 项 | 说明 |
|----|------|
| 上传压缩 | `photoCompressClient.js`：预压缩、流水线、多图并行 |
| 上传进度 | 槽位遮罩 + 全局条 + XHR `onprogress` |
| AI 队列 | `server/services/aiRequestQueue.js`；润色 / 卖点 / 描述提取共用排队 |
| MySQL | 连接池 `MYSQL_CONNECTION_LIMIT`（默认 20） |

**环境变量**：见 `.env.example` 中 `AI_QUEUE_*`、`MYSQL_CONNECTION_LIMIT`

## 一、新增功能（v1.1 及以前）

### 1.1 长图渲染与导出（PNG）

| 项 | 说明 |
|----|------|
| 输出格式 | 长图统一导出 **PNG**（`resvg` + `sharp`），不再以 SVG 作为分享/保存格式 |
| 预览模式 | `previewMode: true` 时宽度 ≤375px，嵌入图压缩，显著加快模板页预览 |
| 子进程渲染 | `posterRenderWorker.js` + CLI，避免主进程阻塞 |
| 图片内嵌 | 上传照片、二维码以 base64 写入 SVG 再栅格化，解决浏览器跨域/外链无法显示 |
| 车况介绍 | 模板新增 `vehicle_description` 区块，展示 `polishedDescription \|\| extraDescription` |
| 照片网格 | 除主图外展示**全部**上传照片（不再默认限 4 张） |
| 多车分割线 | 区块间显示「编号 {下一台编号}」+ 加粗主题色分割线（72px 高、圆角徽章） |
| 多车首台编号 | 每台车区块顶部独立显示 `编号: xxx` |

**相关文件**：`server/services/posterRender.js`、`posterImageEmbed.js`、`posterRenderWorker.js`、`assets/templates/*.json`

### 1.2 服务端生成记录（不持久化 PNG 文件）

| 项 | 说明 |
|----|------|
| 存储策略 | 服务端**不写**长图 PNG 到磁盘，仅写生成元数据 |
| 数据表 | `poster_generations`、`poster_generation_vehicles`（迁移 `003_poster_generations.sql`） |
| API 响应 | `POST /api/posters/render` 返回 `imageBase64` + `generationId`（预览模式 `generationId` 为 null） |
| 车辆标记 | `hasPoster` / `lastPosterGeneratedAt` 替代原 `longImageUrl` 字段语义 |

### 1.3 终端缓存与图库

| 模块 | 文件 | 说明 |
|------|------|------|
| 长图缓存 | `public/js/posterCache.js` | IndexedDB；key 含模板、车辆 updatedAt、经销商信息 |
| 多车图库 | `public/js/galleryStore.js` | 多车（≥2 台）生成长图后**自动入库** |
| 命名规则 | `YYYYMMDD-NN` | 如 `20260604-01`；同日递增序号 |
| 删除记录 | `galleryStore.deleteGalleryItem`、API | 图库卡片支持「删除记录」；多车删 IndexedDB + 可选删 `poster_generations`；单车删服务端生成记录并清本地缓存 |

### 1.4 二维码清晰嵌入（可扫描）

| 项 | 说明 |
|----|------|
| 编码 | 二维码单独走 `fileToQrcodeDataUri`：**PNG 无损** + **最近邻**缩放 |
| 像素 | 按模板 `qrcodeSize` × 导出倍率 × 4 计算目标边长（正式 ≥480px） |
| 上传 | 经销商二维码上传要求 **≥200×200**，避免过小图放大模糊 |
| 对比 | 普通实拍仍 JPEG 压缩，不影响二维码 |

**相关文件**：`server/services/posterImageEmbed.js`、`posterRender.js`
| 去重 | 15 分钟内同 vehicleIds + templateId 更新条目，不重复新增 |
| 图库 Tab | `renderGallery()` | 先展示多车合集，再展示单车已生成长图条目 |

### 1.4 上传与录入体验

| 项 | 说明 |
|----|------|
| 一次选多张 | 顶部「一次选多张」按 **外观→内饰→座椅** 共 13 槽自动分配 |
| 编辑替换 | 「✏️ 编辑」进入拍照页，加载已有照片；再次点击槽位**替换**而非无限追加 |
| 卖点默认 | 推荐卖点**默认不选中**，需用户手动勾选 |
| ~~识别刷新~~ | v1.2 已移除识图流程 |
| 卖点带入描述 | v1.2 改为点选卖点实时 `mergeSellingIntoDescription` 融入正文 |

### 1.5 经销商资料与分享

| 项 | 说明 |
|----|------|
| 资料维护 | 「我的」页编辑姓名/店铺、电话、微信号；`PUT /api/dealer` |

### 1.6 移动端与部署

## 二、Bug 修复清单

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | 长图预览/分享中照片不显示 | SVG `<image>` 引用 `/uploads/` 外链，浏览器限制 | 照片 base64 内嵌后再栅格化 |
| 2 | 车况介绍不显示 | 无 `vehicle_description` 渲染类型 | 新增区块 + 三模板配置 |
| 3 | 长图只显示部分/最后一张图 | `photo_grid` maxCount 限制；部分模板缺网格 | 默认展示除主图外全部照片 |
| 4 | 上传多图只剩最后一张 | Multer `filename` 回调读不到 `req.body` 中 slot | 内存接收 + UUID 独立存储；FormData 字段顺序修正 |
| 5 | 朋友圈分享预览失败 | 分享面板无预览区；路径字段不一致 | 新增预览区；统一 `posterDataUrl` |
| 6 | 推荐卖点无法带入描述 | 描述页未合并卖点 | `buildDescFromSellingPoints()` |
| 7 | 姓名/电话/二维码无法维护 | 无编辑接口与上传 API | `PUT/POST /api/dealer*` + 「我的」表单 |
| 8 | 生成长图姓名电话未更新 | 未保存即渲染 | `persistDealerIfNeeded()` 自动同步 |
| 9 | 编号遮挡二维码 | 底部固定 280px，编号与二维码重叠 | 动态高度 + 编号移至二维码下方 |
| 10 | 多车第一台无编号 | `vehicle_code` 在 SKIP 集合中 | 移除跳过；每车区块顶部渲染编号 |
| 11 | 长图保存为 SVG | 分享/下载扩展名与格式不符 | 统一 PNG 栅格化输出 |
| 12 | 长图内二维码模糊难扫 | 二维码与普通图同走 JPEG 压缩 | `fileToQrcodeDataUri` PNG + 最近邻；上传 ≥200px |
| 13 | 图库无法删除生成记录 | 多车卡片无删除；单车仅可删整车 | 「删除记录」+ `DELETE /api/vehicles/:id/poster-record` |
| 14 | 上传无进度反馈 | fetch 无进度事件 | 槽位遮罩 + 全局条 + XHR `upload.onprogress` |
| 15 | 长图预览偏慢 | 每次预览 fork 子进程 | 预览模式主进程内联 `renderPosterToBuffer` |
| 16 | 长图嵌入耗时长 | 大图 JPEG 1600px 全并行 | 预览 480px/正式 1200px、并发池、fastShrinkOnLoad |
| 17 | 预览长图缺图 | 预览 `gridMaxCount` 默认仅 4 张 | 移除预览张数上限；嵌入路径与模板 `photo_grid` 对齐 |
| 18 | 「我的」已发布数不更新 | 统计按生成次数；发布后未刷新 profile | `countPublishedVehicles` + 发布后 `await updateProfile()` |

## 三、性能与架构优化

| 优化项 | 策略 | 效果 |
|--------|------|------|
| 预览速度 | 375px 预览 + sharp 压缩嵌入 + 预览不写库 | 模板页切换模板/车辆时响应更快 |
| 渲染隔离 | 子进程 CLI 渲染 | 避免大图生成阻塞 HTTP 主线程 |
| 终端缓存 | IndexedDB 按内容 key 缓存 | 同参数重复进入模板页免请求 |
| 预览 debounce | 280ms 防抖 | 减少连续 API 调用 |
| 请求取消 | `AbortController` | 快速切换模板时取消过时预览 |

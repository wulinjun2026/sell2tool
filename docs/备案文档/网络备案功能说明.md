# 通用产品销售助手 — 系统功能及细节亮点说明

> 适用于网站/应用网络备案时的「服务内容说明」「功能简介」等材料撰写参考。可根据备案主体实际名称、域名、联系方式等自行替换括号内占位内容。

---

## 一、系统概述

**系统名称**：通用产品销售助手  
**版本号**：v1.1  
**系统类型**：B/S 架构 Web 应用（支持 PWA 安装至手机桌面，可选 Android 壳应用）  
**访问方式**：通过浏览器访问（如：http://【备案域名】）  
**主要用途**：面向个体商户、小微销售从业者，提供**产品信息录入、图片管理、AI 辅助文案、营销长图生成与本地分享**等工具型服务，帮助用户高效制作适合微信朋友圈等渠道传播的产品宣传长图。

**服务性质说明**：本系统为**自用型业务工具**，不提供公开社交、论坛、直播、新闻资讯、在线交易支付、用户间即时通讯等功能；用户数据按账号隔离，仅服务账号持有人自身的产品管理需求。

## 二、主要功能模块

### 1. 用户账号与权限

| 功能 | 说明 |
|------|------|
| 手机号登录 | 用户通过 11 位手机号 + 短信验证码登录，验证码有效期 5 分钟，获取间隔 60 秒 |
| 身份鉴权 | 采用 JWT 令牌机制，接口请求需携带有效令牌，未登录用户无法访问业务数据 |
| 数据隔离 | 不同用户的产品、照片、长图记录、联系信息相互隔离，互不可见 |
| 免费试用 | 新用户注册后享有 **20 天免费试用**，试用期内最多创建 **40 个产品** |
| 付费升级 | 试用到期或达到产品上限后，系统提示升级；升级后解除试用与数量限制（账号体系支持 paid 计划） |

### 2. 产品信息管理

| 功能 | 说明 |
|------|------|
| 产品录入 | 支持新建产品，填写名称、售价、描述等信息 |
| 唯一编号 | 系统自动生成唯一产品编号（格式：CC + 日期时间 + 序号），便于检索与管理 |
| 状态管理 | 支持「在售 / 已售 / 草稿」三种状态，可筛选、搜索、批量标记已售或批量删除 |
| 列表统计 | 首页展示总产品数、在售数、已售数，支持关键词搜索与多选批量操作 |

### 3. 结构化图片上传

| 功能 | 说明 |
|------|------|
| 分步上传 | 引导用户按「外观 / 细节 / 补充」三个步骤上传产品照片，降低漏拍风险 |
| 多槽位管理 | 各步骤下设多个固定拍摄位（如左前、右后、细节特写等），支持单张替换与删除 |
| 批量选图 | 支持「一次选多张」，系统自动按顺序填充至各拍摄位，提升录入效率 |
| 上传进度 | 大文件上传时显示进度反馈；客户端对图片进行压缩优化后再上传，节省流量与存储 |
| 图库浏览 | 独立图库页集中展示已上传产品图片，便于回顾与复用 |

### 4. AI 辅助文案

| 功能 | 说明 |
|------|------|
| 卖点生成 | 根据用户输入的产品描述，由 AI 自动生成多条核心卖点标签，供点选使用 |
| 卖点融入 | 用户点选卖点后，系统自动将要点自然融入产品描述正文，无需手动拼接 |
| AI 润色 | 对产品描述进行语言润色，提升宣传文案的专业度与可读性 |
| 描述预览 | 实时预览含卖点与售价信息的完整产品介绍，所见即所得 |
| 智能降级 | 未配置云端 AI 密钥时，系统自动使用本地规则模板，保障基础功能可用 |
| 请求队列 | AI 相关接口采用排队机制，避免并发过高导致服务不稳定 |

### 5. 营销长图生成

| 功能 | 说明 |
|------|------|
| 模板选择 | 提供多种长图排版模板，用户可选择风格后一键生成 |
| 单车长图 | 单件产品生成完整宣传长图，含主图、细节图网格、产品描述、售价及底部联系信息 |
| 多车长图 | 支持勾选多件产品合并生成超长宣传图，区块间自动插入编号与分割线 |
| 联系信息 | 「我的」页可配置姓名/店铺名、电话、微信号及二维码，自动嵌入长图底部 |
| 预览模式 | 模板选择阶段提供低分辨率快速预览（约 375px 宽），缩短等待时间 |
| 高清导出 | 正式生成时输出高清 PNG 长图，适合保存至相册或分享 |
| 进度反馈 | 生成过程分阶段显示进度（加载模板、嵌入图片、渲染、导出等），98% 后若仍在处理则细粒度动画至 99.9%，与实际阶段同步 |
| 本地缓存 | 已生成长图在浏览器 IndexedDB 中缓存，再次查看或分享无需重复生成 |
| 客户端渲染 | 服务端返回 SVG 结构与图片地址，PNG 栅格化在用户终端完成，减轻服务器压力并提升响应速度 |

### 6. 分享与保存

| 功能 | 说明 |
|------|------|
| 保存长图 | 支持下载 PNG 至本地，或通过系统原生「保存到相册」能力（Android 壳） |
| 分享功能 | 优先调用系统 Web Share API 分享 PNG 图片文件；不支持时提供保存引导 |
| 分享文案 | 可自动生成默认分享文案，用户复制后配合长图发布至微信朋友圈等渠道 |
| 发布记录 | 保存或确认分享后，系统记录发布状态，「我的」页统计已发布产品数量 |

### 7. 个人中心（「我的」）

| 功能 | 说明 |
|------|------|
| 数据统计 | 展示总产品、在售、已发布数量 |
| 试用/配额 | 展示免费试用剩余天数与产品配额使用情况 |
| 联系信息 | 维护长图底部展示的商户联系方式与二维码 |
| 退出登录 | 支持安全退出，清除本地登录令牌 |

## 三、技术架构与细节亮点

### 3.1 整体架构

- **前端**：HTML5 + CSS3 + 原生 JavaScript（ES Module），移动端优先响应式设计
- **后端**：Node.js + Express RESTful API
- **数据库**：生产环境 MySQL；本地开发支持 SQLite
- **部署**：Nginx 反向代理 + systemd 守护进程，支持 Linux 服务器一键部署
- **终端形态**：Web 浏览器 / PWA 渐进式 Web 应用 / Capacitor Android 壳（可选）

### 3.2 性能与体验亮点

1. **上传流水线优化**：图片压缩与上传并行处理，多图预压缩，跳过已优化 JPEG，显著缩短上传耗时。
2. **长图嵌入并行化**：SVG 内多图 base64 嵌入采用 6 路并发 + LRU 缓存，多图场景生成速度明显提升。
3. **前后端分工渲染**：服务端负责排版结构与 SVG 组装，终端负责图片嵌入与 PNG 导出，兼顾质量与服务器负载。
4. **AI 请求队列**：润色、卖点生成等 AI 调用统一排队（可配置并发数与队列上限），防止峰值压垮服务。
5. **连接池管理**：MySQL 连接池可配置上限，适应多用户并发访问。
6. **PWA 离线壳**：Service Worker 缓存静态资源，支持添加至主屏幕，接近原生 App 使用体验。

### 3.3 安全与合规设计

1. **鉴权全覆盖**：除健康检查、静态资源外，业务 API 均需登录令牌。
2. **验证码安全**：短信验证码哈希存储，5 分钟过期；登录令牌 HMAC 签名，30 天有效期。
3. **数据隔离**：所有产品、照片、长图记录按 `user_id` 严格隔离。
4. **文件上传限制**：照片上传经 Multer 中间件，限制文件类型与大小，存储于用户专属目录。
5. **无 UGC 公开传播**：用户生成的内容仅本人可见与管理，系统不提供公开内容广场或用户间内容流转。

## 四、适用场景与用户群体

**适用场景**：

- 个体商户、微商、小微销售从业者日常产品宣传
- 需要在微信朋友圈、微信群等私域渠道发布图文广告
- 多 SKU 产品需要批量制作统一风格宣传长图

**目标用户**：具备基本智能手机操作能力的成年商户用户，需使用本人手机号注册登录。

## 五、系统不涉及的功能（备案说明常用）

为便于主管部门审核，明确说明本系统**不包含**以下功能：

- ❌ 互联网新闻信息服务
- ❌ 网络出版、视听节目、直播
- ❌ 网络社交、论坛、博客、即时通讯
- ❌ 在线支付、网络购物、金融理财
- ❌ 网络游戏、网络音乐、网络视频平台
- ❌ 公开 UGC 内容社区或信息聚合展示
- ❌ 用户间私信、评论、点赞等社交互动（分享至微信由用户在本机完成，非系统内社交功能）

## 六、数据存储说明

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 用户账号 | 服务器数据库 | 手机号、注册时间、套餐计划等 |
| 产品信息 | 服务器数据库 | 名称、描述、售价、状态、编号等 |
| 产品照片 | 服务器文件系统 | 按用户与产品分目录存储 |
| 长图生成记录 | 服务器数据库 | 记录生成时间、关联产品，不持久化 PNG 文件 |
| 长图缓存 | 用户终端 IndexedDB | 本地缓存，加速再次查看 |
| 登录令牌 | 用户终端 localStorage | 仅保存在用户设备 |

用户可申请注销账号及删除数据（需运营方配合实现，备案材料中可注明「按用户申请处理」）。

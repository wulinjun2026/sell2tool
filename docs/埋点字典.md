## 1. 设计原则

| 原则 | 说明 |
|------|------|
| 隐私优先 | 不上报相册路径、原图内容；vehicle_id 可用匿名 UUID |
| 可离线 | 事件先入本地队列，联网后批量上报（二期） |
| 对齐 OKR | 覆盖发布耗时、分享率、复用率等 RPD §8 指标 |
| 接口抽象 | 经 `AnalyticsPort` 注入，便于 Mock / 切换厂商 |

**MVP**：可先仅本地日志 + Debug 面板；生产接入 Firebase / 友盟 / 自建。

### 3.2 车辆录入（对应 page-upload / page-desc）

| 事件名 | 触发时机 | 专属属性 |
|--------|----------|----------|
| `vehicle_create_start` | 点击「+ 录入」 | `entry: tab \| list_button` |
| `vehicle_created` | 首次保存获得 code | `vehicle_id`, `code` |
| `upload_step_view` | 进入上传某步 | `step: 1\|2\|3`, `step_name: exterior\|interior\|seats` |
| `upload_step_completed` | 点击跳过或进入下一步 | `step`, `photos_count`, `skipped: boolean` |
| `photo_added` | 成功添加照片 | `vehicle_id`, `category`, `slot_key`, `source: camera\|gallery`, `count_in_slot` |
| `photo_removed` | 删除照片 | `vehicle_id`, `slot_key` |
| `photo_edited` | 裁切/滤镜/水印完成 | `vehicle_id`, `edit_type: crop\|filter\|watermark` |
| `desc_page_view` | 进入描述页 | `vehicle_id` |
| `ai_polish_requested` | 点击 AI 润色 | `vehicle_id`, `scene: vehicle_description\|selling_points`, `raw_length` |
| `ai_polish_success` | 润色完成 | `vehicle_id`, `source: llm\|local_template`, `duration_ms`, `output_length` |
| `ai_polish_failed` | 润色失败 | `vehicle_id`, `error_code` |
| `ai_polish_accepted` | 用户选润色版 | `vehicle_id` |
| `ai_polish_rejected` | 用户保留原文 | `vehicle_id` |
| `selling_points_generate_requested` | 点击「AI 生成卖点」 | `raw_length`, `has_brand` |
| `selling_points_generate_success` | 卖点生成完成 | `source: llm\|local_template`, `count`, `duration_ms` |
| `selling_points_generate_failed` | 卖点生成失败 | `error_code` |
| `selling_point_selected` | 勾选/取消卖点（描述页） | `vehicle_id`, `point_id`, `selected: boolean` |
| `vehicle_draft_saved` | 录入流中途保存草稿 | `vehicle_id`, `completion_rate: number` |

**completion_rate**：已填槽位数 / 总槽位数（0~1）。

---

### 3.3 长图生成（page-template）

| 事件名 | 触发时机 | 专属属性 |
|--------|----------|----------|
| `poster_generate_start` | 点击生成长图 | `vehicle_ids: string[]`, `vehicle_count`, `template_id` |
| `poster_generated` | 生成成功 | `vehicle_ids`, `vehicle_count`, `template_id`, `duration_ms`, `output_width`, `output_height`, `file_size_bytes`, `is_multi` |
| `poster_generate_failed` | 生成失败 | `vehicle_ids`, `error_code`, `duration_ms` |
| `poster_preview_scroll` | 预览滑动（采样 10%） | `scroll_depth_ratio: number` |
| `template_selected` | 切换模板 | `template_id`, `from_template_id` |
| `poster_saved_to_album` | 保存相册 | `vehicle_ids`, `file_size_bytes` |

**error_code 枚举**：`OOM`, `TIMEOUT`, `NO_PHOTOS`, `TEMPLATE_INVALID`, `STORAGE_FULL`

---

### 3.4 分享（share-bridge）

| 事件名 | 触发时机 | 专属属性 |
|--------|----------|----------|
| `share_panel_open` | 打开分享面板 | `vehicle_ids`, `is_reuse: boolean` |
| `share_copy_edited` | 用户修改预填文案 | `vehicle_ids`, `edit_length_delta` |
| `share_confirm` | 点击分享到朋友圈 | `vehicle_ids`, `share_type`, `photo_extra_count` |
| `share_success` | 微信回调成功 | `vehicle_ids`, `share_type`, `is_reuse`, `duration_from_generate_ms` |
| `share_cancelled` | 用户取消 | `vehicle_ids` |
| `share_failed` | 分享失败 | `vehicle_ids`, `error_code` |

**share_type**：`long_image_only` | `long_image_with_photos` | `photos_only`

**is_reuse**：`true` = 本地已有长图直接分享（OKR：复用率）

## 4. 性能计时（Timing Events）

通过 `trackTiming(name, durationMs, props)` 上报：

| name | 测量区间 | 目标（RPD） |
|------|----------|-------------|
| `poster_render_duration` | generate_start → generated | 单车 ≤3000ms，5车 ≤8000ms |
| `ai_polish_duration` | requested → success | — |
| `cold_start_duration` | launch → 首屏可交互 | — |
| `photo_compress_duration` | 选图 → 写入完成 | — |

---

## 5. OKR 指标映射

| OKR（RPD §8） | 计算方式 |
|---------------|----------|
| 单次发布耗时 ≤30s | `share_success.duration_from_generate_ms` 且 `vehicle_count=1` 的 P50 |
| 多车发布 ≤60s | 同上 `vehicle_count>=2` 的 P50 |
| 长图保存率 ≥80% | `poster_saved_to_album` + `share_success` / `poster_generated` |
| 朋友圈分享率 ≥60% | `share_success` / `poster_generated` |
| 本地复用率 ≥50% | `share_success.is_reuse=true` / `share_success` |
| 次日留存 ≥40% | 厂商标准留存报表（需 `user_id` 稳定） |

## 6. 漏斗定义

### 6.1 新建发布漏斗

```
vehicle_create_start
  → upload_step_completed (step=3)
  → desc_page_view
  → selling_points_generate_success（可选）
  → poster_generated
  → share_success
```

### 6.2 AI 润色漏斗

```
ai_polish_requested
  → ai_polish_success
  → ai_polish_accepted
```

## 9. 隐私与合规清单

- [ ] 不上报：相册 EXIF 精确 GPS、手机号明文、原图 base64
- [ ] `user_id` 使用随机 UUID，不与微信 OpenID 混用除非明示授权
- [ ] 提供设置项「加入体验改进计划」关闭上报（中国区建议默认关或二次确认）
- [ ] 隐私政策中列出埋点厂商与字段范围

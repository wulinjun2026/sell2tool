## 1. 产品背景

二手车经销商在朋友圈/微信群发布车辆信息时，面临以下痛点：

- 图片需手动裁切、排版，耗时较长
- 车辆卖点需手动撰写，缺乏专业文案
- 发布内容形式单一，传播效果差
- 无法一键生成适合社交传播的长图
- 生成长图后仍需手动保存再切换到微信分享，流程割裂
- 无法一次发布多台车辆，需要逐台操作效率低
- 已发布车辆长图无法本地留存，再次分享须重新制作
- 已售车辆无标记管理，容易重复发布

本产品旨在提供 **「拍照 → 描述与 AI 卖点 → 多车生成长图 → 一键分享朋友圈 → 本地管理复用」** 的闭环体验。

---

## 2. 产品定位

一款面向二手车销售顾问和车商的 **移动端信息发布效率工具**，帮助用户在 30 秒内生成单张或多车超长图，并支持已发布车辆的本地留存、复用与分享管理。

---

## 3. 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 二手车销售顾问 | 每天需发布 5~10 条车源信息 | 快速出图，突出卖点 |
| 车商老板/店长 | 管理多台车辆发布 | 统一模板，品牌输出 |
| 二手车平台运营 | 批量制作车源素材 | 标准化内容生产 |
| 门店管理员 | 管理多台车辆发布/下架 | 车辆列表管理、已售标记删除 |

## 4. 功能需求

### 4.1 本地图片上传（结构化流程）

按照固定流程分步指引用户拍摄/上传照片，每一步均为非必选，且支持追加照片。

**上传流程结构：**

```
第一步：外观（Exterior）             第二步：内饰（Interior）          第三步：座椅（Seats）
  ├─ 前（车头正面）                     ├─ 中控                        ├─ 前排座椅
  ├─ 后（车尾正面）                     ├─ 车机（屏幕/仪表）             ├─ 后排座椅
  ├─ 45度左（左前45°）                  └─ 主驾驶位                    ├─ 后备箱
  ├─ 左（左侧面）                                                            └─ 前备箱（如有）
  ├─ 45度右（右前45°）
  └─ 右（右侧面）
```

**分步上传界面：**

| 步骤 | 分类 | 子项 | 说明 |
|------|------|------|------|
| 1 | 外观 | 前、后、45度左、左、45度右、右 | 每项至少 1 张，可追加多张（追加无上限）；支持从相册选择或拍照 |
| 2 | 内饰 | 中控、车机、主驾驶位 | 每项非必选，可追加多张 |
| 3 | 座椅 | 前排座椅、后排座椅、后备箱、前备箱 | 每项非必选，可追加多张；前备箱仅新能源/前置行李箱车型显示 |

**交互规则：**

| 功能 | 说明 |
|------|------|
| **固定顺序** | 严格按照 外观 → 内饰 → 座椅 三步引导上传，不可跨步跳转 |
| **非必选** | 每一步每一项均为可选，用户可跳过任意子项或整个步骤 |
| **追加照片** | 每个子项在已有照片基础上支持继续追加（「+」按钮唤起相册/相机） |
| **一次选多张** | 顶部「一次选多张」从相册批量选择，按 **外观→内饰→座椅** 顺序自动填入 13 个槽位 |
| **编辑替换** | 从列表「编辑」进入拍照页时加载已有照片；再次点击已有槽位可**替换**该位置照片 |
| **照片预览** | 每个子项以缩略图网格展示已上传照片，支持点击大图查看 |
| **删除单张** | 缩略图支持长按/点击「×」删除单张 |
| **已完成标记** | 已上传照片的子项显示 ✅ 角标，未上传的显示「+」占位 |
| **进度提示** | 顶部进度条显示总完成度（如 外观 3/6 · 内饰 1/3 · 座椅 1/4） |
| **跳过步骤** | 每步底部提供「跳过此步，进入下一步」按钮 |
| **拍照/相册** | 每个子项均支持系统相机拍照或从相册选择 |
| **图片裁切** | 上传后提供 1:1 / 4:3 / 16:9 比例裁切 |
| **滤镜/调色** | 一键增强（提亮、去雾、饱和度优化），可选 3~5 个汽车专属滤镜 |
| **水印** | 支持添加经销商名称/联系方式水印，位置可拖拽 |

**步骤进度条示意：**

```
步骤 1/3：外观 ────────────────────────
         ● 前  ✅  ● 后  ✅  ● 45度左  +  ● 左  +  ● 45度右  +  ● 右  +
         [跳过此步 → 内饰]

步骤 2/3：内饰 ────────────────────────
         ● 中控  +  ● 车机  +  ● 主驾驶位  +
         [跳过此步 → 座椅]

步骤 3/3：座椅 ────────────────────────
         ● 前排座椅  +  ● 后排座椅  +  ● 后备箱  +  ● 前备箱  +
         [跳过此步 → 完成]
```

### 4.1.1 文本描述车辆属性

照片上传完成后，进入车辆属性文本描述页面，支持自由输入车辆其他信息，并可使用大模型进行润色优化。

| 功能 | 说明 |
|------|------|
| **属性描述** | 提供多行文本输入框，用户可自由描述车辆配置、车况、维修记录、改装情况等任意属性 |
| **输入引导** | 输入框上方显示提示文案，如「补充车辆其他亮点，如: 全程4S店保养、隐形车衣、加装电尾门...」 |
| **车型手填** | 描述页顶部提供车型输入框（不再自动识图） |
| **售价录入** | 描述页提供售价（万元）输入，预览与长图车况介绍末尾追加 `售价：X万元` |
| **卖点并入** | 点选/取消卖点后，以「正文，卖点1，卖点2，…！」自然融入描述（无 `【车辆亮点】` 独立块） |
| **实时预览** | 描述页 `#desc-preview` 展示车况介绍预览（含已选卖点与售价），与长图 `vehicle_description` 一致 |
| **AI 生成卖点** | 点击「AI 生成卖点」，根据描述/车型/售价由大模型生成标签；「换一批」重新生成 |
| **大模型润色** | 点击「AI 润色」按钮，将用户输入的原始文本发送至大模型，返回润色后的专业文案 |
| **润色风格** | 大模型按二手车销售场景优化：修正语病、增强说服力、自动插入 Emoji、控制篇幅在 100 字以内 |
| **对比模式** | 润色后展示「原文 vs 润色文」左右对比，用户可选择保留原版或使用润色版 |
| **多次润色** | 支持对同一文本多次调用大模型润色，每次生成不同版本 |
| **手动编辑** | 润色后的文案支持继续手动微调 |

**交互流程：**

```
照片上传完成
    ↓
进入文本描述页面
    ↓
用户在输入框中自由输入车辆描述
    ↓
点击「AI 润色」按钮
    ↓
大模型返回润色文案（带加载动画）
    ↓
对比展示：原文 | 润色版
    ↓
用户选择：使用润色版 / 使用原文 / 再次润色
    ↓
确认后进入模板选择页
```

> **v1.2 变更**：原独立「卖点推荐页」（`page-sell`）已移除，卖点生成与选择全部在描述页完成。

### 4.2 车型卖点推荐（并入描述页）

| 功能 | 说明 |
|------|------|
| **车型录入** | 用户手动填写车型；不再提供上传后自动识图 |
| **LLM 生成卖点** | `POST /api/selling-points/generate`：根据 `rawText`、`brandModel`、`priceWan` 生成 12 条以内标签 |

| **默认不选中** | 生成/推荐标签**默认不选中**，由用户手动点选 |
| **卖点融入描述** | 点选后调用 `mergeSellingIntoDescription`：正文与卖点以逗号衔接，句末统一标点 |
| **去重剥离** | 取消点选或重新生成时，`stripKnownSellingTexts` 从正文中移除已知卖点文案 |
| **卖点分类** | appearance、interior、performance、value、resale、maintenance |
| **Emoji 增强** | 每条卖点带 Emoji 标签展示 |

| 功能 | 说明 |
|------|------|
| **车型列表勾选** | 展示已录入车辆列表，支持复选框同时勾选多台车辆 |

### 4.4 一键分享到朋友圈

| 功能 | 说明 |
|------|------|
| **原生分享** | 长图/超长图生成后直接唤起微信朋友圈分享面板，无需先保存到相册 |

**分享流程（新建）：**

```
长图/超长图生成 → 预览确认
    ↓
编辑分享文案（系统预填 + 用户微调）
    ↓

    ↓

    ↓
唤起微信分享面板 → 用户确认发布
    ↓
发布成功后自动保存车辆及长图到本地
```

**分享流程（复用）：**

```
打开本地车辆列表
    ↓
选择已保存车辆
    ↓
  ┌─ 直接分享 ──→ 唤起微信分享面板（无需重新生成）
  │
  └─ 修改后分享 ──→ 编辑卖点/价格/文案 → 重新生成 → 分享
    ↓
发布成功后更新本地保存
```

### 4.5 车辆唯一编号系统

每台录入车辆自动生成全局唯一编号，贯穿车辆全生命周期（草稿→在售→已售），并展示在长图中。

| 功能 | 说明 |
|------|------|
| **自动编码** | 车辆首次保存时自动生成唯一编号，不可修改 |
| **编码规则** | `CC + yyyyMMdd + HHmm + 3位顺序号`，如 `CC202606021430001` |
| **编码组成** | 前缀 `CC`（CarCar）+ 录入日期 `yyyyMMdd` + 录入时间 `HHmm` + 当日顺序号 `001~999` |
| **时间可读** | 编码中 `20260602` = 2026年6月2日，`1430` = 14:30，一眼可识别录入时间 |
| **顺序递增** | 同一天内按录入顺序递增末3位（001, 002, 003...），次日重置从001开始 |
| **显示位置** | 车辆详情页顶部、长图右上角、列表卡片底部以灰色小字展示 |
| **扫码溯源** | 长图中二维码扫码后可查看车辆详情页，顶部展示完整编号 |
| **查重保障** | 编码全局唯一，避免同一车辆重复录入 |

**编码格式详解：**

```
CC  20260602  1430  001
 │     │       │     │
 │     │       │     └── 当日顺序号（001~999）
 │     │       └──────── 录入时间（24小时制）
 │     └──────────────── 录入日期（yyyyMMdd）
 └────────────────────── 固定前缀（CarCar）
```

### 4.6 车辆管理与本地图库

| 功能 | 说明 |
|------|------|
| **车辆列表** | 以卡片列表展示所有已录入车辆，每车显示缩略图、唯一编号、车型、年份、里程、售价、状态标签 |
| **状态标签** | 车辆状态包括「在售」「已售」「草稿」，列表页可直观区分 |
| **本地长图留存** | 已生成长图的车辆标记 `hasPoster`；客户端 IndexedDB 缓存 PNG Base64；**服务端不保存 PNG 文件** |
| **多车图库** | 勾选 ≥2 台车生成长图后，**自动存入本地图库**，名称 `YYYYMMDD-NN`（如 `20260604-01`）；15 分钟内同批次更新不重复新增 |
| **图库展示** | Tab「图库」先展示多车合集条目（名称、台数、缩略图），再展示单车已生成长图卡片 |
| **再次修改** | 已保存车辆支持修改图片、卖点、价格、文案，修改后重新生成长图 |
| **直接分享** | 已保存车辆点击「分享」直接唤起微信朋友圈，跳过生成流程 |
| **已售标记** | 车辆售出后标记为「已售」，列表中以灰色/角标区分 |
| **已售删除** | 已售车辆支持一键删除，释放本地存储空间；删除前二次确认 |
| **批量管理** | 支持多选车辆，批量标记已售/批量删除 |
| **搜索/筛选** | 按车型、年份、价格区间、状态（在售/已售/草稿）筛选车辆 |

## 5. 业务流程

### 5.1 新建发布流程

```
开始
  ↓
录入车辆信息（图片/车型/卖点/价格） → 系统自动生成唯一编号
  ↓
进入车辆列表（列表展示编号） → 勾选单台或多台车辆
  ↓
选择「生成长图」
  ↓
选择长图模板 ─────→ 实时预览
  ↓
满意？──→ 否 ──→ 调整内容/模板
  ↓ 是
一键分享到朋友圈 ──→ 编辑文案 → 唤起微信 → 发布成功
  ↓（备选）
保存到相册 / 分享到其他平台
  ↓
车辆及长图自动保存至本地图库
  ↓
结束
```

### 5.2 车辆管理与复用流程

```
打开本地图库
  ↓
查看车辆列表（在售/草稿/已售）
  ↓
选择已保存车辆
  │
  ├─ 直接分享 ──→ 唤起微信分享面板
  │
  ├─ 修改后分享 ──→ 编辑信息 → 重新生成长图 → 分享
  │
  ├─ 标记已售 ──→ 状态变更为「已售」
  │
  └─ 删除车辆 ──→ 确认删除 → 释放本地存储
  ↓
更新车辆状态
```

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| **性能** | 单车型长图生成 ≤ 3 秒；多车超长图（5台以内）≤ 8 秒；预览模式 ≤375px 宽优先响应 |
| **超长图加载** | 预览降采样 + 终端 IndexedDB 缓存；正式分享前生成高清 PNG |
| **图片压缩** | 预览嵌入图 sharp 压缩；正式 PNG 导出；单张超长图 ≤ 8MB |
| **兼容性** | iOS 13+ / Android 8+；H5 PWA + Capacitor Android 壳 |
| **网络** | 卖点库、模板、编号、长图渲染可离线降级（LLM 除外） |
| **存储** | 服务端：照片 `/uploads/` + 生成记录；客户端：IndexedDB 长图缓存（24 条）+ 多车图库（100 条） |
| **隐私** | 相册原图存服务端 uploads（当前 Web MVP）；长图 PNG 不在服务端持久化 |

## 9. 附录

### 9.1 竞品参考

- **车300**：车型识别 + 估价，缺少内容生成
- **懂车帝商家版**：有发车功能，长图样式单一
- **Canva/稿定设计**：模板丰富，但非汽车垂直领域，操作门槛高

### 9.2 技术选型建议

| 模块 | 方案 |
|------|------|
| 图片处理 | CoreImage (iOS) / RenderScript (Android) / Canvas (Web) |
| 长图渲染 | 原生 Canvas（分块渲染，避免 OOM）；超长图采用异步分块拼接 + 渐进式加载 |
| 本地存储 | Realm / SQLite + 文件系统（图片 LRU 缓存，上限 500MB） |
| 车型识别 | 本地模型 TensorFlow Lite / ONNX |
| 卖点推荐 | 本地规则引擎 + 关键词匹配 |
| 大模型润色 | 接入云端 LLM API（如 DeepSeek / GPT），流式输出润色结果；支持离线降级到本地规则模板 |
| 超长图加载优化 | 虚拟列表（VirtualScroll）思想：仅渲染可视区域区块，滑动时按需加载/卸载图片 block；大图使用缩略图占位 + 原图渐进替换 |
